This release makes the platform Secure by Design regarding asset handling.

  • SECURITY Strict Tenant Isolation: The system now forcefully blocks any file operation that attempts to write outside a tenant's designated sandbox (tenants/{id}/...). This eliminates the risk of cross-tenant data leakage via file uploads.
  • FEATURE Runtime Architecture Watchdog: A new "Watchdog" service continuously monitors the server's file system. It instantly detects and alerts if any "legacy" folder structures (like user-photos in root) are created, ensuring the architecture remains clean over time.

Breaking Change for Legacy Code:

Any code attempting to save files to identifying "Legacy Paths" (e.g., /site-logo, /products-photos) or referencing paths without a valid Tenant ID will now throw a SecurityException or IllegalArgumentException.

Action: Ensure all file operations occur within a valid TenantContext and use the standard path structure: tenants/{id}/assets/{type}/...
